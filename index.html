Reference

API Guide

GET

/api/hello

Basic test endpoint that passes through the rate-limit filter.

GET

/admin/stats?clientId=...

Returns rate-limit stats for one known client ID.

POST

/admin/generate-key?clientName=...

Creates a new API key and maps it to the provided client name.

DELETE

/admin/revoke-key?apiKey=...

Revokes an existing API key.

Headers

How this frontend talks to the backend

Optional header for tracked clients:
X-API-Key: rl-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

If X-API-Key is missing or invalid:
the backend identifies the client by IP address.

Recommended integration flow:
1. Generate an API key
2. Use that key in your application requests
3. Query /admin/stats with that same key as clientId
